### 大模型显存空间浪费

- 提前申请过多空间（内部碎片）

- 过小无法分布给其他请求（外部碎片）

-  

- PagedAttention

	- 借鉴操作系统的虚拟内存和分页技术来管理注意力计算中的KV缓存

		- 分块存储

			- 将KV cache划分为固定大小的块(block),类似操作系统的页

			- 每个块存储固定数量token的KV向量(如16个token)

			- 块可以非连续存储,通过块表(block table)映射逻辑位置到物理位置

		- 按需分配

			- 只在生成新token时才分配新块

			- 避免预先分配大量可能用不到的内存

			- 序列生成结束后立即释放块供其他请求使用

		- 内存共享

			- 对于相同的prompt前缀,多个请求可以共享KV cache块

			- 采用Copy-on-Write机制:只读共享,写时复制

			- 特别适合批量推理和beam search场景

	- 工作原理

		- 根据块表找到每个token的KV向量所在的物理块

		- 从不连续的块中读取KV数据

		- 执行标准的注意力计算

		- 核心insight:注意力计算本身不要求KV在内存中连续

	- 性能提升

		- 内存利用率: 提高到接近100%,传统方法仅20-40%

		- 吞吐量:内存利用率提升使得批处理大小增加2-4倍

		- 延迟:更大批处理带来更好的GPU利用率

### 大模型推理推理二阶段

- prefill阶段

	- 输入的prompt做self-attention时可以并行计算

	-  

- decode阶段

	- 输出的token，无法并行计算，只能一个个token生成

	-  

- 显存占用

	-  

	- Memory=batchsize×seqlength×hiddensize×layers×2×2
（序列长度、向量长度、层数，FP16 一个数 2 个字节， kv 2个）

	- Prefilling 阶段由于输入 prompt 的 token 数已经确定，所以在显存中我们可以直接按照所需要的矩阵大小申请显存空间

	- 对于 Decode 阶段，我们并不知道什么时候会停止生成，也不知道要输出多少个 token 才会见到我们想见到的「EOS」符，而我们又需要保证 KVcache 在显存中的连续性

- vllm显存优化

	- 虚拟显存（逻辑显存地址连续=》碎片物理显存地址）

	- batch策略

		- LLM 服务显存占用主要由三部分组成：模型权重、KVCache 以及其他激活值。而现有的 LLM 服务中，KVCache 和激活值显存开销与推理时的 batch size 成正比，batch size 越大，KVCache 和激活值显存开销越大

		- KVCacheMemory=2×batch_size×num_layers×max_seq_len×(hidden_size/num_heads)×num_heads×sizeof(dtype)

		- 优点

			- 多条数据，只需加载一次模型参数进显存，提高吞吐

		- 传统策略

			-  

		- 动态batch（continue batchding）

			- 处理过程中，如果某条请求处理完了，就踢出去，然后腾出空间放下一条请求数据

## 大模型微调

### 传统大模型微调（Bert,T5,Bart）

- 1. 微调所有参数

- 2. 调整部分参数(冻结部分层)

- 3. 对模型尾部加一些层，训练新加层

### 调参

- Batchsize=10， 6b参数，需要显存：6*6b*10G=360G显存，所以一般只训练部分层（如倒数2层，共2kw参数）

	- 显存里面到底存了哪些数据
(显存：n*6（估计常量）*6b）

		- 模型：6b 个 float32 浮点数

		- Adam 梯度下降法：存前向结果，存之前步骤的梯度结果

		- 上面总体估计为n*6b*常量6

	- 因为显存的限制

		- 模型可训练参数不能太多

		- Batchsize不能太大 

		- Max_length 不能太大

			- 经验：
1. 一般会统计平均长度（去除异常值，特别短和特别长的）
2. 平均长度*1.5
3. 确定好训练参数
4. Batchsize理论上来说，越大越好（从大到小尝试，直到可以跑，尽量榨干gpu性能）

### 微调模型参数，使用场景

- 1. 数据量大 token数量>=可调参数，保证效果

- 2. 机器算力足 显存大小决定能否训练 显卡flops决定训练时间 

- 3. 场景非常垂直

### 大模型调参难点，为什么很少直接微调

- 1. 参数多，显存不容易放下

- 2. 参数多，需要对应更大数据

- 3. 参数多，不容易收敛

- 4.参数多，调参时间过长

### 参数高效微调方法（Parameter-Efficient Fine-Tuning，PEFT）三件套

- Prefix-Tuning / Prompt-Tuning：在模型的输入或隐层添加 k 个额外可训练的前缀 tokens（这些前缀是连续的伪 tokens，不对应真实的 tokens），只训练这些前缀参数

	- Prompt-Tuning

		- 问题：今天天晴，温度25度。今天能否打篮球 

	- Prefix-Tuning：Prefix+问题

		- 问题：a b c d e今天能否打篮球 答案：今天可以打篮球 Prompt加真实提示词，Prefix加虚拟token

- Adapter-Tuning：将较小的神经网络层或模块插入预训练模型的每一层，这些新插入的神经模块称为 adapter（适配器），下游任务微调时也只训练这些适配器参数

-  LoRA：通过学习小参数的低秩矩阵来近似模型权重矩阵 W的参数更新，训练时只优化低秩矩阵参数

### 参数类型

- 待训练参数要用float32，如果用float16 可能会出现 nan inf 

## transform

###  

- 由一个多抽头self-attention和一个FFN前馈网络组成

- 利用了resnet的模式，解决了快速收敛和梯度问题

- 有encode和decode两种模式，encode可以看到全部信息，decode可以看到部分信息

- 红色画圈部分用于信息融合（非必须），广泛用于多模态，机器翻译等场景

- Attention的作用获取上下文，FFN的作用是存储知识

- Normal用的是LN(layer normal)，不是BN(batch normal)

	- LN和BN区别

		- 都是用于对数据进行正规化，将输入数据归一至正态分布，加速收敛，提高训练的稳定性

		- BN：一个batch的向量，同一维度的数据做正规化

		- BN的缺点在于：变长数据无法处理，语义数据无法处理，所以有了LN

		- 对于BN，在训练时，是对每一批数据进行归一化，也即用每一批数据的均值和方差；而在测试时，比如进行一个样本的预测，并没有batch概念，因此用的均值和方差是全量训练数据的均值和方差，这个可以通过移动平均法求得

		- BN算法防止过拟合：在网络的训练中，BN的使用使得一个minibatch中所有的样本都被关联在一起，因此网络不会从某一个训练样本中生成确定的结果，即同一个样本的输出不再仅仅取决于样本本身，也取决于和这个样本同属一个batch的其他样本，而每次网络都是随机取batch，这样就会使的网络不会朝一个方向使劲学习，一定程度上避免了过拟合

		- LN：序列向量中，不同时刻的向量做正规化

		- LN不依赖于batch的大小和sequence的长度，因此可用于batchsize为1和RNN中的sequence的normalize操作。padding会对batch normalization的seq len这个维度有影响

- 运算量大B^2, 加速算法kv-cache

- post-norm与pre-norm

	-  

		- 下图中LN和F不能互换位置，因为LN是正态分布，数值范围比较小，如果更换的话，那主导权将由上面的路径主导

	- post-norm在残差之后做归一化，对参数正则化的效果更强，进而模型的鲁棒性更好

	- pre-norm相对于post-norm，因为有一部分参数直接加在了后面，不需要对这部分参数进行正则化，正好可以防止模型的梯度爆炸或梯度消失

	- 如果层数少，post-norm的效果要好一些，如果把层数加大，为了保证模型的训练，pre-norm显然更好一些

- multi head self attention中为啥QKV要用三个不同的矩阵

	- 使用不同的矩阵，反映了将同一向量映射到不同语义空间，从而提升了模型的语义表达能力

	- 如果使用相同的矩阵，相同量级的情况下，q和k进行点积的值会是最大的，进行softmax的加权平均后，该词所占的比重会最大，使得其他词的比重很小，无法有效利用上下文信息来增强当前词的语义表示，而使用不同的qkv后，会很大程度减轻上述影响

### 分词器

- BPE编码

	- 词表构建后，需要对训练语料进行编码，编码流程如下：
1. 将词表中的单词按照长度大小，从长到短进行排序
2. 对于语料中的每个单词，遍历排序好的词表，判断词表中的单词/子词（subword）是否是该字符串的子串，如果匹配上了，则输出当前子词，并继续遍历单词剩下的字符串
3. 如果遍历完词表，单词中仍然有子字符串没有被匹配，那我们将其替换为一个特殊的子词

- 词表设计

	- 词表太小：
1. 无法有效表征语义，增大训练难度
2. 序列过长

	- 词表太大：
1. 分类任务难，且慢
2. 长token出现次数如果不过多，训练不充分
3. 需要配合较大的embedding

### [显存分析](https://yuanchaofa.com/post/llm-train-infer-memoery-usage-calculation.html)

- 模型训练2个过程

	- b 代表 batch size，s 代表 sequence length，L 代表层数，h 代表隐变量大小，a 代表 attention 头数

		- batch size太小：学习中的抖动，不稳定

		- batch size太大：占用显存，容易陷入局部极小，缺乏探索性

	- 前向计算（计算loss）

		-  

			- attention 结构

				- QKV 矩阵相乘，我们只需要存储它们共享的大小为 2sbh 的输入

				- QK^T，需要同时存储 Q 和 K，总共需要 4sbh 的存储空间

				- softmax，反向传播需要大小为2as^2b的 softmax 输出

				- softmax dropout，只需要as^2b的 mask 大小

				- attention*V，我们需要存储 dropout 的输出（2as^2b）和 Values（2sbh），因此需要2as^2b+2sbh的存储空间

				- 最后的 linear projection 将其输入激活存储为大小为 2sbh，而注意力 dropout 需要大小为 sbh 的掩码

				- 将上述值相加，总体而言，注意力块需要11sbh+5as^2b字节的存储空间

			- FFN结构

				- 两个线性层分别使用大小为 2sbh 和 8sbh 的输入进行存储

				- GeLU 非线性函数在反向传播时还需要大小为 8sbh 的输入

				- dropout 使用大小为 sbh 的掩码进行存储

				- FFN 结构总共需要 19sbh 的存储空间

			- Layer Norm

				- 每个 Layer Norm 操作都使用大小为 2sbh 的输入进行存储，因此总共我们需要 4sbh 的存储空间

			- 因此加起来整体的显存占用量，乘上层数是 sbhL(34+5as/h)（单位：byte）

	- 反向传播（更新梯度）

		-  

			- 前向传播用FP16，更新参数用FP32

			- 全量微调：至少需要16倍模型大小的显存，如
1b的模型，需要16G的显存

	- 训练总的显存

		- 16Φ + 34sbhL+5baLs^2

	- 推理的显存

		- 以半精度BF16为例，模型的参数为θ，那么推理时候占用的显存是 2Φ

		- KV cache，以 float16 来保存 KV cache，那么 KV cache 的峰值显存占用大小为 b(s + n)h * l * 2 * 2 = 4blh(s + n) 。这里第一个 2 表示 K/V cache，第二个 2 表示 float16 占 2 个 bytes

		- 粗略的预估，模型推理需要的内存为：1.2 倍的模型参数内存 = 1.2 × 2Φ = 2.4Φ，以 7B 模型为例，那么推理需要的内存大概是：16.8 GB。 相对精确的预估：按照上面的公式进行计算

				- 整体流程（上层）

					- 输入：原始图像 + 提示（mask/points/box/text 等，告诉模型 “分割哪里” 

					- image encoder（图像编码器）：提取图像的全局特征（image embedding），把像素信息转化为高维语义特征（如 256×64×64 的特征图，通道数 256，空间尺寸 64×64 ）

					- prompt encoder（提示编码器）：把多样化提示（点、框、文本等）编码成提示特征（prompt tokens ），让模型理解 “分割需求”

					- mask decoder（掩码解码器）：融合图像特征 + 提示特征，预测出目标掩码（mask，即分割结果 ），并输出置信度（score ）

				- 关键模块详解（下层 mask decoder 核心逻辑）

							- 作用：让提示特征内部互相交互，整合多提示信息（比如同时给 “点 + 文本” 提示，自注意力让它们协同工作 ）

							- 原理：提示特征自己做 Query/Key/Value，挖掘提示间的关联

						- token to image attn.（提示→图像注意力）

							- 作用：让提示特征“反向关注”图像特征，细化提示与图像的关联（比如提示框可能不够准，通过图像反馈调整关注区域 ）

							- 原理：用提示特征做 Query，图像特征做 Key/Value，进一步校准关联

						- mlp（多层感知机）

							- 作用：在注意力交互后，用 MLP 对特征做非线性变换，增强特征表达（把注意力输出的线性组合，转化为更复杂的语义特征 ）

						- image to token attn.（图像→提示注意力）

							- 作用：让图像特征“关注”提示特征，找到图像中与提示相关的区域（比如提示是 “剪刀”，图像特征聚焦到剪刀位置 ）

							- 原理：用图像特征做 Key/Value，提示特征做 Query，通过注意力计算（点积），让图像特征向提示对齐

					- 掩码生成层（右侧）

							- 作用：把 “转换后的图像特征” 与 “提示特征” 做点积，得到初始掩码（mask），初步标出目标区域

							- 作用：对生成的掩码预测置信度（IoU scores，交并比评分 ），判断分割结果是否准确（分数越高，分割越准 ）
